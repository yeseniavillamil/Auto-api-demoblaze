Hallazgos de Pruebas Exploratorias con Postman
Seguridad de Contraseñas:
Las contraseñas codificadas en base64 son vulnerables a ataques Man-In-the-Middle.
Recomendar el uso de cifrado más seguro (llave pública/privada).
Códigos de Estado:
Las APIs devuelven código 200 con mensajes de error, lo cual puede dificultar integraciones futuras.
Información Excesiva:
Las respuestas de las APIs revelan información innecesaria, exponiendo datos de usuarios.
Conclusiones de la Prueba Técnica
Cypress vs. Karate:

No hay una herramienta mejor que otra; la elección debe basarse en el contexto.
Elección: Se eligió Cypress por experiencia previa en JavaScript y automatización Web E2E.
Experiencia Adicional:

Familiaridad con API testing usando Rest-Assured y Serenity-BDD.
Interés en explorar Karate para expandir habilidades.
En resumen, la prueba automatizada en Cypress demostró ser efectiva para evaluar servicios API en Demoblaze, con observaciones sobre mejoras en seguridad y estándares de respuesta. La elección de Cypress se basó en experiencia previa y adaptabilidad al contexto.
